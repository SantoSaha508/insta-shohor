Q-1: How does JavaScript work?

JavaScript is a scripting language for the web. We also know it is an interpreted language. Interpreted means it does not need any compiler to translate its code. JavaScript has a single Call Stack. That's why it can do one thing at a time. It is a Client-side Scripting Language.it is a computer programming language that runs inside an Internet browse.

Q-2: What does JS event loop do?

Event loop keeps a good communication with Call Stack and Callback Queue. it constantly checks whether or not the call stack is empty. It checks if the call stack is free, then lets know the callback queue. Then Callback queue passes the callback function to Call stack to be executed. When all the callback functions are executed, the call stack is out and global execution context is free.

Q-3: Local storage vs session storage difference

Local Storage:
1.Data is shared between all tabs and windows from the same origin.
2.The data will not delete.
3.Data will remain even after browser restart and survive OS reboot too.
Session storage:
1.It exists only within the current browser tab.
2.Another tab with the same page will have a different session storage.
3.The data survives page refresh, but not closing/opening the tab.
